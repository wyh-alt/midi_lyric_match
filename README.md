# MIDI 分析与歌词匹配工具

本工具是一个功能强大的 Windows 桌面应用程序，旨在帮助音乐制作人、音频工程师以及 MIDI 爱好者快速分析 MIDI 文件，实现歌词与 MIDI 的自动对齐写入，并在波形上手动校准歌词时间轴。程序采用 Windows 11 Fluent Design 风格设计。

## 📦 运行环境

- **操作系统**：Windows 10 / Windows 11
- **免安装**：已打包为独立的 `exe` 应用程序，无需配置 Python 环境，下载/拷贝到任意 Windows 电脑上双击即可直接运行。

> **提示**：由于采用了单文件打包技术，程序首次启动时需要临时解压资源，可能需要等待数秒钟，请耐心等待。

## 🚀 主要功能

### 1. 文件处理（MIDI 歌词自动对齐与匹配）

将文本格式的歌词文件自动吸附到 MIDI 音符上，并生成带有歌词事件的新 MIDI 文件。

- **支持拖拽操作**：输入框支持直接拖入文件或文件夹；多路径可用分号 `;` 分隔。
- **单文件与批量处理**：
  - **单文件模式**：拖入 MIDI 与歌词文件，自动匹配处理。
  - **批量处理模式**：拖入两个文件夹，按文件名前缀 ID 自动配对批量导出。
- **多路径选择**：MIDI / 歌词输入支持「选文件」「选目录」，并显示已选文件数量。
- **自定义对齐参数**：
  - **吸附容差 (毫秒)**：歌词与 MIDI 音符的最大允许时间误差。
  - **目标轨道**：选择写入轨道（支持 auto 自动识别）。
  - **对齐参照算法**：逐句首字差值 / **全局平均差值**（默认）。
  - **写入前清除原有事件**、**自动填充延音 `-`**、**逐字匹配**。
- **歌词文本过滤**（减少和声/Rap 误吸附）：
  - 可选过滤括号内文本，如 `(伴唱)`、`【和声】`。
  - 可配置忽略关键词整句（逗号分隔，如：过滤,伴唱,和声,合唱,说唱）。

### 2. 歌词时间轴打点校准（新）

在音频波形上对照听歌，手动微调歌词时间，并导出为原格式歌词文件。

- **拖拽加载**：拖入音频文件 + 歌词文件，自动解析波形与歌词行。
- **低延迟播放**：基于 `sounddevice`，优先 **ASIO** / WASAPI，输出设备尽量与 Windows 系统默认一致。
- **波形操作**：`Ctrl + 滚轮` 以鼠标位置为中心缩放；点击/拖动定位播放头。
- **歌词编辑**：
  - 拖拽橙色标记调整单行时间；双击空白处将最近歌词吸附到该位置。
  - **右键框选**多行歌词，`Ctrl+点击` 加选/取消，`Ctrl+A` 全选，批量拖动移动。
  - **过滤锚点**（`|←` / `→|`）：标记不参与导出的时间段；`Delete` 删除悬停锚点。
  - **撤销 / 重做**：覆盖歌词移动、删除、锚点放置与删除等操作。
- **导出**：按源文件格式保存（LRC / 小灰熊 KSC / 通用 TXT 等）；KSC 按句起止时间与字级毫秒数正确回写；`.txt` 扩展名也会根据内容自动识别 KSC/LRC。
- **首次加载引导**：首次成功加载后弹出操作说明（含程序风格示意图）。

### 3. MIDI 分析

深度解析单个 MIDI 文件的内部事件结构。

- **可视化数据表格**：展示音符、歌词、文本等事件。
- **精确时间轴**：轨道、毫秒/秒、音高、力度、文本内容等。

## 📖 使用指南

### 场景 A：给一个 MIDI 文件注入歌词

1. 打开软件，进入 **「文件处理」**。
2. 拖入 **MIDI** 与 **歌词** 文件，选择输出路径。
3. 按需调整对齐参数（默认已为「全局平均差值」）。
4. 点击 **「开始匹配」**，在输出目录查看新 MIDI。

### 场景 B：批量处理多首歌曲

1. 将 MIDI 文件夹与歌词文件夹分别拖入对应输入框（或选目录）。
2. 确保同名前缀一致（如 `01-midi.mid` 与 `01-lyric.lrc`）。
3. 选择输出文件夹，点击 **「开始匹配」**。

### 场景 C：在波形上校准歌词时间

1. 切换到 **「歌词时间轴打点校准」**。
2. 拖入 **音频** + **歌词** 文件，点击加载。
3. 播放对照波形，拖拽/框选调整时间，设置过滤锚点后 **保存**。
4. 导出文件保留原格式，时间戳为校准后的结果。

### 场景 D：检查 MIDI 是否写入歌词

1. 进入 **「MIDI 分析」**，拖入 MIDI，点击 **「开始分析」**。
2. 在表格中核对歌词事件与音符时间戳。

## ❓ 常见问题 (FAQ)

- **Q: 为什么生成的 MIDI 里有些字没匹配上？**  
  A: 歌词与音符时间差超过「吸附容差」时可调大容差；也可换用「全局平均差值」算法。

- **Q: 程序双击后没有反应？**  
  A: 单文件 EXE 首次解压需 3–10 秒，请勿重复双击。

- **Q: 中文歌词乱码？**  
  A: 建议将歌词文件保存为 **UTF-8**；程序会尝试自动识别编码。

- **Q: 校准页听不到声音或延迟大？**  
  A: 在工具栏打开 **音频设备设置**，优先选择 ASIO 或 WASAPI 设备，并与 Windows 默认输出一致。

- **Q: 小灰熊 KSC 导出时间不对？**  
  A: 程序会识别 `karaoke.add` 格式；句结束时间 = 句起始 + 各字毫秒数之和，移动句首后整句同步平移。
